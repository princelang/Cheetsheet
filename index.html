CSS Selector and Combinators Cheet Sheet

CSS Selectors

CSS selectors are used to define a pattern of the elements that you want to select for applying a set of CSS rules on the selected elements. Combinators define the relationship between the selectors. Using various selectors and combinators, you can precisely select and style the desired elements based on their type, attributes, state, or relationship to other elements.

Types of Selectors

There are over 80 selectors and combinators. CSS selectors can be grouped into the following categories based on the type of elements they can select.

Selector Description CSS Syntax CSS Example
Universal Selects all elements. * { style properties } * { box-sizing: boarder-box; }
Type Selects all elements that have the given node name. element { style properties } p { background-color: skyblue; }
Class Selects all elements that have the given class attribute. .class_name { style properties } .header { color: black; }
ID Selects an element based on the value of its id attribute. #id_value { style properties } #info { font-family: proxima nova, sans-serif; }
Compound Combining a type or universal selector with a class or id selector. element#id_value { style properties } a.selected { }
Attribute Selector Selects elements based on the element having a given attribute explicitly set. [attr]{ style properties }
[attr=value]{ style properties }		 a[href=“https://example.com”] { color: green; }
Pseudo-classes Selects a special state of the selected element. :disabled { style properties } input[type="text"]:disabled { background: gray; }

CSS combinators

A complex selector is a sequence of one or more simple and/or compound selectors that are separated by combinators, including the white space which is a descendant combinator.

Combinators Description CSS Syntax CSS example
Descendant Combinator Selects nodes that are descendants of the first element. (denoted with one or more spaces) selector1 selector2 { style properties } li li { list-style-type: circle; }
Child Combinator Selects nodes that are direct children of the first element. (Denoted with the greater than character (>) selector1 > selector2 { style properties } nav > button { background-color: black; }
Grandchild Combinator Selects nodes that are direct grandchildren of the first element. (Denoted with the asterisk ( * ) selector1 * selector2 { style properties } nav * a { color: white; }
Subsequent-Sibling Combinator Selects elements based on their siblings. Aka They have to share a parent. (Denoted with the tilde ( ~ ) former_element ~ target_element { style properties } p ~ a { color: red }
Next-Sibling Combinator Selects elements based on their siblings that was immediately preceded. (Denoted with the plus ( + ) former_element + target_element { style properties } p + a { font-weight: bold; }
Column Combinator Selects elements based on their siblings that was immediately preceded. (Denoted with the plus ( + ) column-selector || cell-selector { style properties } col.selected | | td { background: gray; }
